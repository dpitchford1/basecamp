# Frontend Helpers

This document covers the utility modules that enhance frontend output and admin UX without being large enough for their own dedicated doc: the announcement bar, contextual subnav, page conditionals, image class normalisation, and the various `RemoveBloat` additions.

---

## Announcement Bar (Toast)

**File:** `inc/frontend/class-basecamp-toast.php`  
**Class:** `Basecamp\Frontend\Toast`  
**Global:** `the_toast()`  
**Config:** Appearance → Theme Settings → Announcement Bar

A dismissable banner rendered at the top of every frontend page. Controlled entirely from the settings page — no code changes needed to enable or update it.

### Configuration

| Setting Key | Purpose |
|---|---|
| `toast_enabled` | `'1'` to show the bar, `'0'` to hide |
| `toast_text` | Message text. Bar does not render if empty |
| `toast_url` | Optional URL — wraps the text in an `<a>` tag. Leave empty for text-only |

### Dismissal

Dismissed state is stored in `localStorage` under a key derived from an MD5 hash of the bar's content (`text + url`). Editing the text or URL regenerates the hash and resets dismissed state for all visitors automatically.

### Usage in Templates

```php
<?php the_toast(); ?>
```

Place in `header.php` just inside `<body>`, before the site header.

## Contextual Subnav

**File:** `inc/frontend/basecamp-subnav.php`  
**Global:** `the_subnav()`

Outputs a contextual secondary navigation list for page hierarchies. The logic is automatic — no arguments needed:

- If the current page **has child pages** → lists those children
- If the current page **is a child** → lists its siblings (the other children of its parent)
- If neither condition applies (standalone page with no children) → outputs nothing

### Usage in Templates

```php
<?php the_subnav(); ?>
```

## Page Conditional Helpers

**File:** `inc/frontend/basecamp-page-helpers.php`

### `basecamp_is_page_or_child_of( $slugs_or_ids )`

Returns `true` if the current page **is** the specified page, or is a **descendant** of it at any depth.

```php
// True on /about/ and on /about/team/, /about/team/leadership/, etc.
if ( basecamp_is_page_or_child_of( 'about' ) ) {
    // ...
}

// Accepts an array to match any of several parents
if ( basecamp_is_page_or_child_of( [ 'about', 'services' ] ) ) {
    // ...
}

// Also accepts post IDs
if ( basecamp_is_page_or_child_of( 42 ) ) {
    // ...
}
```

Useful for conditional CSS classes on `<body>` or `<main>`, showing/hiding sidebar elements, or any logic that should apply to an entire section of the page hierarchy.

## Image Class Normalisation

**Method:** `Basecamp\Frontend\Frontend::normalize_img_tag_classes()`  
**Hook:** `wp_get_attachment_image_attributes` at priority 20

WordPress appends `size-{name}` and `attachment-{name}` classes to every `<img>` generated by `wp_get_attachment_image()`. This filter deduplicates all classes on the element and, by default, strips those two WP-generated class groups.

### Opt In to Keeping WP Size Classes

```php
// Preserve size-* and attachment-* classes on all attachment images
add_filter( 'basecamp_keep_wp_image_size_classes', '__return_true' );
```

Default is `false` — the classes are stripped unless explicitly opted in.

## RemoveBloat Additions

**File:** `inc/frontend/remove-bloat.php`  
**Class:** `Basecamp\Frontend\RemoveBloat`

In addition to the standard WP cruft removal (emoji, jQuery Migrate, feed links, REST link, generator, heartbeat), the following behaviours are active:

### `remove wpautop from excerpts`

`wpautop` is unhooked from `the_excerpt`. This prevents WordPress from wrapping excerpt text in `<p>` tags, which is often redundant when the excerpt is already inside a styled card element.

### `hide_admin_bar_for_subscribers()`

Hides the WordPress admin bar for any logged-in user who does **not** have the `edit_posts` capability. This covers the Subscriber role and any custom role without editing rights — they see the site as a regular visitor.

### `conditionally_load_cf7()`

Contact Form 7's scripts and styles (`contact-form-7`, `wpcf7-swv`) are dequeued on every page request. They are re-enqueued only on pages whose slug is in the `basecamp_cf7_page_slugs` filter — which returns an **empty array by default**.

To enable CF7 assets on specific pages:

```php
add_filter( 'basecamp_cf7_page_slugs', function( array $slugs ): array {
    $slugs[] = 'contact';
    $slugs[] = 'get-a-quote';
    return $slugs;
} );
```

No CF7 assets are loaded anywhere until this filter is used.

## Admin Enhancements

### View in Browser — Admin Bar Node

**Method:** `Basecamp\Admin\Admin::add_view_in_browser_node()`

When editing a post or page (`/wp-admin/post.php?action=edit&post=N`), an **↗ View in browser** link is added to the admin bar. It opens the live permalink in a new tab.

The node only appears when:
- The current admin screen is a post edit screen
- The post has a published status (i.e. a viewable permalink exists)
- The current user has `edit_post` capability for that post

## Analytics Filters

**File:** `inc/theme-functions/basecamp-analytics.php`

Two filters control GA4 resource hint behaviour for non-production environments:

| Filter | Default | Purpose |
|---|---|---|
| `basecamp_ga_hints_on_nonprod` | `false` | Set to `true` to output `preconnect`/`dns-prefetch` hints on non-production environments (they are suppressed by default to avoid unnecessary outbound connections during development) |
| `basecamp_ga_preload_enabled` | `false` | Set to `true` to add a `<link rel="preload">` for the gtag script on production. Only active when `is_prod_host()` also returns `true` |

```php
// Example: enable resource hints on staging for performance testing
add_filter( 'basecamp_ga_hints_on_nonprod', '__return_true' );
```
