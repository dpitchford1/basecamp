# Images & Media

This document covers the full image pipeline â€” from upload-time processing through to frontend output â€” including registered image sizes, automatic WebP conversion, srcset handling, and opt-out mechanisms.

---

## Registered Image Sizes

All custom sizes are registered in `inc/class-basecamp.php` via `add_image_size()` inside the theme setup hook. WordPress generates every size automatically when an image is uploaded.

### Image Sizes â€” 16:9, Soft Crop

All sizes use soft crop (aspect ratio preserved). Used for hero images, banners, and any context where the image should not be cropped.

| Handle | Width | Height | Notes |
|---|---|---|---|
| `basecamp-img-xl` | 1400 | 800 | Standard desktop full-width |
| `basecamp-img-lg` | 1280 | 720 | Large desktop |
| `basecamp-img-m` | 980 | 560 | Tablet landscape / narrow desktop |
| `basecamp-img-sm` | 600 | 343 | Tablet portrait / wide mobile |
| `basecamp-img-s` | 400 | 229 | Mobile |

### Portrait Sizes â€” 3:4, Hard Crop

Hard-cropped to exactly 3:4 ratio. Used with `srcset` / `<picture>` to serve a portrait-oriented crop on mobile devices, where a landscape crop would be too wide and short to be useful.

| Handle | Width | Height |
|---|---|---|
| `portait-sm` | 300 | 400 |
| `portait-m` | 640 | 853 |
| `portait-lg` | 960 | 1280 |

> **Note:** The handle spelling `portait` (single `r`) is intentional â€” matches the existing database metadata for uploaded images. Do not rename.

### Using Image Sizes in Templates

The simplest output â€” WordPress auto-generates `srcset` from all matching registered sizes. The WebP filter replaces every URL transparently, so no extra code is needed for WebP.

```php
// Featured image at a specific size â€” srcset generated automatically
the_post_thumbnail( 'basecamp-img-lg' );

// Any attachment
echo wp_get_attachment_image( $attachment_id, 'basecamp-img-lg' );

// Custom sizes attribute (override the default WordPress guess)
echo wp_get_attachment_image( $attachment_id, 'basecamp-img-lg', false, [
    'sizes' => '(max-width: 600px) 100vw, (max-width: 1280px) 50vw, 33vw',
    'class' => 'hero__img',
    'loading' => 'eager', // above-fold images should be eager
] );

// URL only â€” WebP filter applies here too
$src = wp_get_attachment_image_src( $attachment_id, 'basecamp-img-xl' );
echo esc_url( $src[0] );
```

## Responsive Images â€” Portrait / Landscape `<picture>`

**Helper:** `Basecamp_Frontend::picture()`

For contexts where a portrait crop should be served on mobile and a landscape image on wider screens, use the `picture()` helper. It outputs a `<picture>` element with an optional portrait `<source>` (single URL, one mobile-optimised crop) plus a full-featured landscape `<img>` via `wp_get_attachment_image()`.

Because the landscape element is generated by `wp_get_attachment_image()`, the browser receives:
- A full WordPress-generated `srcset` across all landscape sizes
- Automatic WebP URL substitution (via the filter layer)
- `width` and `height` attributes (prevents layout shift)
- All loading attributes (`loading`, `decoding`, `fetchpriority`)

### Basic Usage

```php
Basecamp_Frontend::picture( get_post_thumbnail_id(), [
    'class'     => 'hero__picture',
    'img_class' => 'hero__img',
] );
```

Outputs:

```html
<picture class="hero__picture">
    <!-- Portrait source: single URL, served at â‰¤ 600px -->
    <source srcset="portait-m.webp" media="(max-width: 600px)">
    <!-- Landscape <img>: full WP srcset, WebP, width/height all automatic -->
    <img
        src="img-xl.webp"
        srcset="img-s.webp 400w, img-sm.webp 600w, img-m.webp 980w, img-lg.webp 1280w, img-xl.webp 1400w"
        sizes="(max-width: 1400px) 100vw, 1400px"
        width="1400" height="800"
        class="hero__img"
        loading="lazy"
        decoding="async"
        alt="Post title"
    >
</picture>
```

> `width`, `height`, and `decoding="async"` are always output â€” no args needed.

### All Arguments

| Argument | Type | Default | Notes |
|---|---|---|---|
| `portrait_size` | `string\|false` | `'portait-m'` | Size handle for the portrait `<source>` URL. `false` skips the `<source>` entirely |
| `portrait_break` | `string` | `'600px'` | `max-width` at which the portrait `<source>` applies |
| `landscape_size` | `string` | `'basecamp-img-xl'` | Size handle passed to `wp_get_attachment_image()` |
| `alt` | `string\|null` | `null` | Alt text. `null` uses the attachment's own alt field. Pass `''` for decorative images |
| `class` | `string` | `''` | Class on `<picture>` |
| `img_class` | `string` | `''` | Class on the `<img>` inside the picture |
| `loading` | `string` | `'lazy'` | `'eager'` for above-fold images |
| `fetchpriority` | `string\|false` | `false` | `'high'` for the first/hero image in a page. `false` omits the attribute |

### Hero / Above-Fold Image

```php
Basecamp_Frontend::picture( get_post_thumbnail_id(), [
    'class'         => 'hero__picture',
    'img_class'     => 'hero__img',
    'loading'       => 'eager',
    'fetchpriority' => 'high',
] );
```

### First Item in a Loop

Use `fetchpriority` and `loading` conditionally on the first card to tell the browser to prioritise it:

```php
$is_first = true;
foreach ( $items as $item ) {
    Basecamp_Frontend::picture( $item['image_id'], [
        'class'         => 'card__picture',
        'img_class'     => 'card__img',
        'loading'       => $is_first ? 'eager' : 'lazy',
        'fetchpriority' => $is_first ? 'high' : false,
    ] );
    $is_first = false;
}
```

### Landscape-Only (no portrait)

Pass `portrait_size => false` to skip the portrait `<source>` â€” useful for landscape-only imagery like banners or horizontal cards where a portrait crop is never needed:

```php
Basecamp_Frontend::picture( $attachment_id, [
    'portrait_size' => false,
    'landscape_size' => 'basecamp-img-lg',
    'img_class'     => 'banner__img',
    'loading'       => 'lazy',
] );
```

Or skip `picture()` entirely and use `wp_get_attachment_image()` directly â€” it generates a full `srcset` automatically with WebP applied:

```php
echo wp_get_attachment_image( get_post_thumbnail_id(), 'basecamp-img-lg', false, [
    'sizes'   => '(max-width: 600px) 100vw, 50vw',
    'class'   => 'card__img',
    'loading' => 'lazy',
] );
```

### How WebP Works Here

The portrait `<source>` URL is resolved via `wp_get_attachment_image_src()`, which is filtered by `basecamp_filter_get_image_src()`. The landscape `<img>` is generated by `wp_get_attachment_image()`, which is filtered by `basecamp_filter_wp_get_attachment_image_attributes()`. Both filters substitute `.webp` URLs automatically when a WebP version exists. No special handling required in templates.

### Hand-Built srcset

For custom markup that needs a srcset string without a `<picture>` wrapper, use `Basecamp_Frontend::build_srcset()`:

```php
$srcset = Basecamp_Frontend::build_srcset( $attachment_id, [
    'basecamp-img-s',
    'basecamp-img-sm',
    'basecamp-img-m',
    'basecamp-img-lg',
] );
// Returns: "img-s.webp 400w, img-sm.webp 600w, img-m.webp 980w, img-lg.webp 1280w"
echo '<img src="..." srcset="' . $srcset . '" sizes="100vw">';
```

### Regenerating Sizes

If you register a new size after images have already been uploaded, existing uploads will not have that size. Use a WP-CLI command to regenerate:

```bash
wp media regenerate --yes
```

Or target a specific attachment:

```bash
wp media regenerate 42 --yes
```

## WebP Conversion

### On Upload â€” Automatic

**File:** `inc/img-optimization/basecamp-webp-conversion.php`

Every JPEG/PNG uploaded to the Media Library is automatically converted to WebP. The hook is `wp_generate_attachment_metadata`, which fires after WordPress has generated all registered image sizes.

What happens on upload:

1. WordPress generates all registered sizes from the original.
2. The `basecamp_convert_uploaded_image()` function fires.
3. It converts the **original file** â†’ `image.jpg.webp`
4. It loops through every size in `$metadata['sizes']` and converts each â†’ `thumb.jpg.webp`

The `.webp` files sit alongside their originals in the same `/wp-content/uploads/` directory. The originals are never deleted â€” they serve as fallback for browsers without WebP support and are used by the admin UI.

Conversion uses the server's best available method (checked in order):

- PHP **GD** with WebP support (`imagecreatefromjpeg` / `imagecreatefrompng` + `imagewebp`)
- PHP **ImageMagick** (`Imagick` extension)
- The **`cwebp`** command-line binary

If none are available, conversion silently skips and only the original is stored. Check **Tools â†’ WebP Conversion** to verify server capability.

### Bulk Conversion Tool

**Admin location:** Tools â†’ WebP Conversion  
**Capability required:** `manage_options`

For images uploaded before this theme was installed (or before the WebP module was active), use the bulk conversion tool. It processes one image per AJAX request to avoid timeouts, tracks progress in two custom database tables, and can be paused and resumed.

**Database tables:**

| Table | Purpose |
|---|---|
| `{prefix}_basecamp_webp_conversions` | Per-image record: status, file path, sizes, conversion time, error |
| `{prefix}_basecamp_webp_stats` | Per-batch summary: totals, space saved, quality setting, timestamps |

Batch progress persists between page loads. A "Reset Conversion" link marks the batch cancelled and resets all non-completed records to `pending` so you can start fresh.

**Quality setting:** Default is `80` (good balance of quality vs file size). Range is 0â€“100. This is stored per batch.

## Frontend WebP Serving

**File:** `inc/img-optimization/basecamp-webp-functions.php`

The frontend layer intercepts every image output and replaces URLs with `.webp` equivalents â€” but only when:

1. A `.webp` version of the image actually exists on disk.
2. The browser is detected as supporting WebP via `basecamp_webp_is_supported()`.

### Browser Support Detection

`basecamp_webp_is_supported()` uses a two-stage check:

1. **Primary â€” `Accept` header:** if `$_SERVER['HTTP_ACCEPT']` contains `image/webp`, return `true`. This is the correct spec-compliant path and works on most stacks.
2. **Fallback â€” User Agent:** if the `Accept` header is absent (can happen on some local stacks or behind certain reverse proxies that strip it), the UA string is checked instead. IE (`MSIE`) and legacy EdgeHTML (`Trident/`) return `false`; all other current engines (Blink, Gecko, WebKit) return `true`.

> **Note:** The UA fallback exists because some local environments (MAMP, Valet, Herd variants) do not forward `Accept` headers from the browser to PHP's `$_SERVER`. Removing the fallback will break WebP serving in those environments.

### Hooks Applied

| WordPress hook | Function | What it does |
|---|---|---|
| `wp_get_attachment_image_src` | `basecamp_filter_get_image_src` | Replaces URL returned by `wp_get_attachment_image_src()` |
| `wp_get_attachment_image_attributes` | `basecamp_filter_wp_get_attachment_image_attributes` | Replaces `src` and every URL in `srcset` |
| `wp_get_attachment_image` | `basecamp_check_webp_conversion` | Re-scans full `<img>` markup for any remaining `src`/`srcset` values |
| `post_thumbnail_html` | `basecamp_filter_post_thumbnail_html` | Replaces `src` in `get_the_post_thumbnail()` output |
| `the_content` (Ă—2) | `basecamp_filter_content_images` + anonymous fn | Replaces `src` in post body images including Gutenberg `wp-image-*` classes |

All filters skip in admin context (except during AJAX calls).

### WebP URL Detection

`basecamp_get_webp_image( $url )` checks for two storage formats:

- **Replacement format:** `image.webp` (replaces original extension)
- **Appended format:** `image.jpg.webp` (appended â€” this is what the conversion module creates)

The function converts the URL to a filesystem path, checks for the `.webp` file, and returns the WebP URL if found. Returns `false` if not found, URL is already `.webp`, or the URL is an SVG.

## Opting Out of WebP

### Per-Element: CSS Classes

Add the `native-img` or `no-webp` class to any `<img>` element to force the original format, regardless of browser support:

```php
echo wp_get_attachment_image( $attachment_id, 'basecamp-img-lg', false, [
    'class' => 'native-img',
] );
```

### Per-Request: URL Query Parameters

Append either of these to any page URL to disable WebP replacement for the entire request (useful during testing):

```
?disable_webp=1
?test_native_srcset=1
```

### Posts with Gallery Shortcode

WebP replacement is automatically suppressed on any post that contains a `[gallery]` shortcode, to prevent conflicts with WordPress's native gallery markup.

### Admin Save Protection

The conversion filter skips execution during `post.php` and `post-new.php` admin page loads (outside of AJAX) to prevent corrupting attachment metadata during saves.

## Adding New Image Sizes

Add new sizes in `inc/class-basecamp.php` inside `setup()`, alongside the existing `add_image_size()` calls:

```php
add_image_size( 'my-custom-size', 800, 600, true ); // width, height, hard-crop
```

Then register the label for the media library insert UI:

```php
function basecamp_custom_image_sizes( $sizes ) {
    return array_merge( $sizes, [
        // ... existing entries ...
        'my-custom-size' => __( '800 by 600', 'basecamp' ),
    ] );
}
```

Run `wp media regenerate --yes` after adding a new size to generate it for existing uploads.
